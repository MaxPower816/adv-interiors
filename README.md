# ADV INTERIORS

Премиальный интерактивный сайт-портфолио дизайнера интерьеров на Next.js, TypeScript, Tailwind CSS, GSAP, Lenis, React Three Fiber, Drei, Framer Motion, React Hook Form и Zod.

## Установка и запуск

```bash
npm install
npm run dev
npm run build
npm run lint
```

## Где менять контент

- Имя, контакты и SEO: `src/config/site.ts`
- Навигация: `src/config/navigation.ts`
- Социальные сети: `src/config/socials.ts`
- Тексты главной и философии: `src/content/site-content.ts`
- Проекты и пути изображений: `src/content/projects.ts`
- Услуги: `src/content/services.ts`
- Прайс: `src/content/pricing.ts`
- Отзывы: `src/content/testimonials.ts`
- FAQ: `src/content/faq.ts`
- Юридические popup-тексты: `src/content/legal.ts`
- Путь камеры по scroll-сцене: `src/config/camera-path.ts`

## Как добавить проект

Добавьте объект в массив `projects` в `src/content/projects.ts`. Для реальных фото положите файлы в `public/images` и замените `cover`, `images`, `layout`.

## Как заменить процедурную комнату на GLB

Инструкция находится в `public/models/README.md`. Коротко: положите модель в `public/models/room.glb` и сохраните имена групп из `src/config/room-stages.ts`. Сайт автоматически использует GLB, если файл существует, и возвращается к процедурной комнате, если файла нет.

## Как настроить timeline мебели

Стадии редактируются в `src/config/room-stages.ts`: `start`, `end`, текст, сторона текстового блока и список объектов.

## Как настроить камеру

Ключевые точки камеры находятся в `src/config/camera-path.ts`.

Каждый объект отвечает за один этап прокрутки:

- `id` — понятное имя этапа.
- `progress` — место на прокрутке от `0` до `1`.
- `position` — где стоит камера `[x, y, z]`.
- `lookAt` — куда камера смотрит `[x, y, z]`.
- `mouse.x` и `mouse.y` — сила реакции на курсор.
- `damping` — плавность догоняния камеры.

Ось `x`: минус — левее, плюс — правее.
Ось `y`: больше — выше, меньше — ниже.
Ось `z`: больше — дальше назад, меньше — ближе вперед.

Чтобы добавить новый этап, вставьте новую точку в массив `cameraPath` и задайте `progress`, например `0.42`.

В первой 3D-сцене включена шаговая прокрутка: один жест колесом или трекпадом ведет к следующей точке из `cameraPath`. Поэтому количество “остановок” сцены задается количеством объектов в `src/config/camera-path.ts`.

Если нужно больше остановок, добавьте новые точки с разными `progress`. Если нужно меньше остановок, удалите лишние точки.

## Camera Studio

На первой 3D-сцене есть dev-панель `Camera Studio` в левом нижнем углу.

Как пользоваться:

1. Откройте `http://127.0.0.1:3011`.
2. Нажмите `Camera Studio`.
3. Выберите этап камеры.
4. Включите `Редактирование ON`.
5. Двигайте сцену мышью: левая кнопка вращает, колесо приближает, правая кнопка или Shift двигает.
6. Нажмите `Взять текущий вид`.
7. Нажмите `Сохранить в файл`.

После сохранения значения записываются в `src/config/camera-path.ts`.

## Интеграции заявки

API route находится в `src/app/api/contact/route.ts`. Сейчас он валидирует данные, защищает honeypot-полем и логирует заявку. Переменные для Telegram, email, CRM и webhook описаны в `.env.example`.

## Деплой на Vercel

Создайте проект на Vercel, добавьте переменные окружения из `.env.example`, выполните стандартный деплой из Git-репозитория. Перед публикацией проверьте `npm run build`.

## Оптимизация GLB и мобильной версии

Используйте Draco или Meshopt, текстуры WebP/AVIF, ограничивайте DPR, отключайте тяжелые тени на слабых устройствах и проверяйте 390/430/768 px. Для слабых устройств предусмотрен fallback и режим `prefers-reduced-motion`.
